# Design: Behavior Detection 前端可视化配置系统

日期: 2026-04-28
状态: DRAFT
模式: Builder (内部工具，从 warehouse-vision 拆分)

## 问题陈述

behavior-detection 项目当前是纯后端 API，从 warehouse-vision 拆出了聚集/打架/跌倒三项检测能力。
缺少前端界面，运维人员无法：
- 实时查看摄像头检测画面
- 在视频画面上可视化配置检测区域（ROI）
- 调整规则参数（聚集人数阈值、打架速度阈值等）
- 查看和过滤历史告警事件

## 核心用户

仓库/园区安防运维人员。需要在浏览器上完成摄像头接入、检测区域配置、规则调参、实时监控、事件查看的完整闭环。

## 已有资产

### 后端（已实现）
- `src/server.py`: FastAPI，已有 `/api/cameras`、`/api/events`、`/api/status`、`/stream/{id}`、`/ws/events`
- `src/analyzer.py`: 每路摄像头一个分析线程，MJPEG 流 + 检测框叠加
- `src/rules/`: 聚集(CrowdRule)、打架(FightRule)、跌倒(FallRule) 三项规则
- `src/config.py`: Pydantic 配置模型（CameraConfig、RulesConfig、AppConfig）
- `configs/default.yaml`: YAML 配置文件

### 后端需补充的 API
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/cameras` | 添加摄像头 |
| PUT | `/api/cameras/{id}` | 更新配置（含 ROI、规则参数） |
| DELETE | `/api/cameras/{id}` | 删除摄像头 |
| GET | `/api/cameras/{id}/snapshot` | 获取当前帧截图（用于 ROI 编辑） |
| PUT | `/api/cameras/{id}/rules` | 单独更新规则参数 |
| GET | `/events/{filename}` | 静态文件，事件截图（已实现） |
| POST | `/api/video-analysis/upload` | 上传视频文件 |
| POST | `/api/video-analysis/tasks/{id}/start` | 配置完成后启动分析 |
| GET | `/api/video-analysis/tasks` | 分析任务列表 |
| GET | `/api/video-analysis/tasks/{id}` | 任务详情（含检测结果） |
| GET | `/api/video-analysis/tasks/{id}/video` | 下载标注视频 |
| GET | `/api/video-analysis/tasks/{id}/first_frame` | 获取首帧（用于画 ROI） |
| DELETE | `/api/video-analysis/tasks/{id}` | 删除任务 |

### 事件截图机制（已实现）
- 后端 `analyzer.py` 检测到事件时，自动保存当前帧截图到 `events/` 目录
- 截图包含：所有检测框（绿色）+ 事件目标高亮（红色）+ 事件类型标签
- 文件名格式：`{camera_id}_{sub_type}_t{track_id}_{timestamp}.jpg`
- 事件对象包含 `image` 字段，值为文件名
- 前端通过 `/events/{filename}` 访问截图
- Events 页表格每行显示缩略图，点击弹出大图预览

### 前端（待实现）
- `web/package.json` 已创建，React 19 + TypeScript + Vite + Tailwind CSS

## 技术选型

| 层 | 技术 | 理由 |
|----|------|------|
| 框架 | React 19 + TypeScript | 与 warehouse-vision 前端一致 |
| 构建 | Vite | 快，warehouse-vision 已验证 |
| 样式 | Tailwind CSS | 轻量，不引入 Ant Design |
| 路由 | react-router-dom v7 | 标准选择 |
| 图标 | lucide-react | 轻量 SVG 图标库 |
| ROI 编辑 | HTML Canvas API | 原生，无额外依赖 |
| 实时通信 | WebSocket (原生) | 后端已有 /ws/events |
| 视频流 | MJPEG (img 标签) | 后端已有 /stream/{id} |

### ROI 可视化编辑器（Config 页核心组件）

**交互流程：**
1. 进入编辑模式，左侧显示摄像头当前帧截图（GET `/api/cameras/{id}/snapshot`）
2. 鼠标点击画面添加多边形顶点，顶点显示为可拖拽的圆点
3. 双击或点击第一个顶点闭合多边形
4. 闭合后，多边形区域半透明填充显示
5. 拖拽顶点可调整位置
6. 右键顶点可删除该顶点
7. [清除] 按钮清除整个 ROI
8. [撤销] 按钮撤销上一步操作
9. 坐标归一化为 0~1 存储（相对于画面宽高）

**技术实现：**
- HTML Canvas 叠加在 img 元素上
- Canvas 尺寸与图片尺寸同步
- 鼠标事件：mousedown（添加/选中顶点）、mousemove（拖拽）、mouseup（释放）、dblclick（闭合）
- 绘制：fillRect 画顶点、strokePath 画边、fillPath 半透明填充区域
- 坐标转换：canvas 像素坐标 ↔ 归一化坐标（0~1）

**边界情况：**
- 窗口 resize 时重新计算 canvas 尺寸和顶点位置
- 截图加载失败时显示 "无法获取画面" + 重试按钮
- 最少 3 个顶点才能闭合
- 防止自交叉多边形（可选，v1 不做）

## 数据流

### 配置保存流程
```
用户在 Config 页编辑 ROI + 规则参数
        │
        ▼
PUT /api/cameras/{id}  ──▶  server.py 接收
        │
        ▼
更新 configs/default.yaml  ──▶  重启对应 CameraAnalyzer 线程
        │
        ▼
新配置生效，检测区域和规则参数立即更新
```

### 事件推送流程
```
CameraAnalyzer 检测到异常
        │
        ▼
调用 push_event()  ──▶  存入内存事件列表
        │
        ▼
广播到所有 WebSocket 客户端  ──▶  Live 页实时显示
```

## 实现优先级

| 优先级 | 任务 | 说明 |
|--------|------|------|
| P0 | 后端 CRUD API | POST/PUT/DELETE cameras + snapshot |
| P0 | 前端项目骨架 | Vite + React + 路由 + 布局 |
| P0 | Live 页 | MJPEG 流 + WebSocket 事件 |

| P2 | 配置热重载 | 修改配置后不重启整个服务 |

## 成功标准

1. 用户能在浏览器上添加摄像头，填入 RTSP 地址，看到实时视频流
2. 用户能在视频画面上拖拽画出检测区域（ROI 多边形）
3. 用户能开关和调参三项检测规则（聚集/打架/跌倒）
4. 检测到异常时，Live 页实时弹出事件通知
5. Events 页能按类型过滤查看历史事件
