# 行为检测参数配置指南

本文档详细说明聚集、打架、跌倒、徘徊四种检测规则的参数含义及场景调优建议。

---

## 一、通用基类参数

所有检测规则共享以下防抖机制：

| 参数 | 类型 | 说明 |
|------|------|------|
| `confirm_frames` | int | 连续确认帧数。条件必须连续满足 N 帧才触发事件，任何一帧不满足则计数器归零，防止瞬间噪声误报 |
| `cooldown` | float | 冷却时间（秒）。触发一次告警后，冷却期内即使条件满足也不重复告警 |
| `schedule` | object | 时间段调度。可配置仅在指定时间段和星期几启用检测，`enabled: false` 表示全天检测 |

**confirm_frames 与 FPS 的关系：** 实际等效时间 = confirm_frames / detect.fps

| detect.fps | confirm_frames=5 等效时间 |
|------------|--------------------------|
| 1 | 5 秒 |
| 3 | 1.67 秒 |
| 5 | 1 秒 |
| 10 | 0.5 秒 |

## 二、聚集检测（Crowd Detection）

### 算法原理

构建人员邻接图 → BFS 搜索连通分量 → 最大聚类人数超阈值时告警。

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用聚集检测 |
| `max_count` | int | `5` | 聚集人数阈值。连通分量中人数 ≥ 该值时触发告警 |
| `radius` | float | `200` | 聚类半径（像素）。两人中心点距离 < 该值则视为同一群体 |
| `confirm_frames` | int | `5` | 连续确认帧数 |
| `cooldown` | float | `60` | 冷却时间（秒） |

---

## 三、打架检测（Fight Detection）

### 算法原理

多人近距离 + 高速运动检测，结合五种过滤/增强机制：同向过滤、相对速度过滤、距离稳定性过滤、关节重叠增强、手腕速度增强。

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用打架检测 |
| `proximity_radius` | float | `150` | 近距离半径（像素）。两人距离超过该值不被视为打架关系 |
| `min_speed` | float | `80` | 最低有效速度（像素/秒）。取身体中心速度和手腕速度的最大值，低于此值不视为暴力运动 |
| `min_persons` | int | `2` | 最少涉及人数 |
| `confirm_frames` | int | `6` | 连续确认帧数 |
| `cooldown` | float | `30` | 冷却时间（秒） |
| `co_move_cos_threshold` | float | `0.7` | 同向行走余弦阈值。两人运动方向余弦相似度 > 该值且相对速度低 → 判为同行，排除误报 |
| `min_relative_speed` | float | `40` | 最低相对速度（像素/秒）。两人相对速度低于该值 → 非对抗性运动 |
| `min_distance_variance` | float | `10` | 最低距离方差（像素²）。两人间距方差低于该值 → 结伴行走，排除误报 |
| `joint_overlap_threshold` | int | `1` | 关节入侵阈值。手臂关节进入对方 bbox 数量达到该值则增强打架判定 |

---

## 四、跌倒检测（Fall Detection）

### 算法原理

两阶段检测：阶段一检测快速倒地过渡（髋部速度 + 骨架角度 + 宽高比变化），阶段二确认持续异常姿态 + 静止不动。

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用跌倒检测 |
| `ratio_threshold` | float | `1.0` | bbox 宽高比（宽/高）阈值。站立时通常 < 0.5（人比较瘦高），超过该值表示可能倒地（人变宽变矮） |
| `min_ratio_change` | float | `0.5` | 帧间宽高比最小变化量。过滤缓慢蹲下、弯腰等正常动作，只有比值突然增大才视为跌倒 |
| `min_y_drop` | float | `20` | 帧间中心点最小下落量（像素）。确保人确实向下坠落而非原地变姿态 |
| `min_hip_velocity` | float | `30` | 髋部最低下落速度（像素/帧）。区分跌倒（快速下坠）和弯腰（缓慢下蹲）。需加载 Pose 模型才生效 |
| `spine_angle_threshold` | float | `45` | 脊柱角度阈值（度）。脊柱与垂直方向夹角 < 该值视为直立状态。需加载 Pose 模型才生效 |
| `inactivity_frames` | int | `3` | 静止确认帧数。跌倒后需保持静止 N 帧才最终确认（排除蹲起动作） |
| `inactivity_threshold` | float | `15` | 静止判定阈值（像素）。帧间运动量 < 该值视为静止 |
| `history_size` | int | `10` | 姿态历史缓冲区大小。保留最近 N 帧数据，用于判断是否从直立转为倒地 |
| `confirm_frames` | int | `5` | 连续确认帧数。跌倒发生快，可适当调低以加快响应 |
| `cooldown` | float | `30` | 冷却时间（秒） |

---

## 五、徘徊检测（Loiter Detection）

### 算法原理

检测人员在 ROI 内长时间停留且位移小的行为。使用底部中心作为锚点，结合惯性计数、停留时长、位移比、总路径综合判断。

### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | bool | `false` | 是否启用徘徊检测 |
| `min_duration` | float | `60` | 最短停留时间（秒）。在 ROI 内至少停留该时间后才开始评估是否徘徊 |
| `max_distance` | float | `150` | 最大位移距离（像素）。与初始位置欧氏距离 ≤ 该值才算徘徊（没走远） |
| `max_displacement_ratio` | float | `0.3` | 最大位移比。净位移/总路径 ≤ 该值 = 来回走动而非穿行。越小要求越"原地打转" |
| `min_total_path` | float | `50` | 最小总路径（像素）。过滤完全静止不动的人（如站岗保安），需有一定移动量 |
| `trajectory_window` | float | `60` | 轨迹分析滑动窗口（秒）。只分析最近该时间段内的轨迹 |
| `inertia` | int | `3` | 惯性帧数。连续 N 帧在 ROI 内才开始计时，防止边缘进出闪烁 |
| `confirm_frames` | int | `5` | 连续确认帧数 |
| `cooldown` | float | `120` | 冷却时间（秒）。徘徊间隔长，避免对同一人反复告警 |

## 六、场景调优总览

### 镜头类型通用规律

| 参数类型 | 广角镜头（视野大，人小） | 窄角镜头（视野小，人大） |
|---------|----------------------|----------------------|
| 距离类参数 | ↑ 增大（像素距离小） | ↓ 减小（像素距离大） |
| 速度类参数 | ↓ 减小（像素速度低） | ↑ 增大（像素速度高） |
| 位移类参数 | ↓ 减小 | ↑ 增大 |
| 人数阈值 | ↑ 增大（覆盖人多） | 保持或减小 |

---

### 场景一：大厅 / 广场（广角）

适用于车站大厅、商场中庭、公共广场等大范围监控。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 8-10 | 人流大，避免常态性聚集频繁告警 |
| 聚集 | `radius` | 300-400 | 广角下人小，物理距离对应像素少 |
| 聚集 | `confirm_frames` | 8-10 | 人群流动性强，需更长时间确认 |
| 打架 | `proximity_radius` | 200-250 | 同上，适配广角比例 |
| 打架 | `min_speed` | 60-80 | 广角下运动像素值小 |
| 打架 | `confirm_frames` | 6-8 | 适度增加确认 |
| 跌倒 | `ratio_threshold` | 1.2-1.5 | 俯视/广角下人显宽，提高阈值 |
| 跌倒 | `min_ratio_change` | 0.3-0.4 | 广角下比值变化幅度小 |
| 跌倒 | `min_y_drop` | 10-15 | 广角下像素位移小 |
| 跌倒 | `min_hip_velocity` | 20-25 | 广角下像素运动值小 |
| 跌倒 | `spine_angle_threshold` | 45 | 保持默认 |
| 徘徊 | `min_duration` | 90-120 | 开放空间停留时间正常较长 |
| 徘徊 | `max_distance` | 250-400 | 广角像素距离小，需放宽 |

---

### 场景二：走廊 / 门口（窄角）

适用于办公走廊、建筑出入口、电梯间等狭窄区域。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 3-5 | 空间小，少量聚集即需关注 |
| 聚集 | `radius` | 100-150 | 窄角下人大，像素距离值大 |
| 聚集 | `confirm_frames` | 3-5 | 人流少，快速确认即可 |
| 打架 | `proximity_radius` | 80-120 | 近景人大 |
| 打架 | `min_speed` | 100-150 | 快步行走在窄角下速度值高，需提高阈值 |
| 打架 | `confirm_frames` | 4-6 | 保持适中 |
| 跌倒 | `ratio_threshold` | 0.8-1.0 | 平视角度站/倒差异明显 |
| 跌倒 | `min_ratio_change` | 0.5-0.6 | 窄角下比值变化幅度大，可用默认或略高 |
| 跌倒 | `min_y_drop` | 20-30 | 窄角下正常像素位移值 |
| 徘徊 | `min_duration` | 30-60 | 走廊不应长时间逗留 |
| 徘徊 | `max_distance` | 80-120 | 窄角像素距离大 |

---

### 场景三：养老院 / 护理中心

重点关注跌倒检测的快速响应。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `enabled` | `false` | 通常不需要聚集检测 |
| 打架 | `enabled` | `false` | 通常不需要 |
| 跌倒 | `ratio_threshold` | 0.8-1.0 | 标准值 |
| 跌倒 | `min_ratio_change` | 0.3-0.4 | 老人动作慢，降低变化阈值更易触发 |
| 跌倒 | `min_y_drop` | 15-20 | 老人体型较小，适当降低 |
| 跌倒 | `min_hip_velocity` | 20-25 | 老人运动较慢，降低速度阈值 |
| 跌倒 | `spine_angle_threshold` | 50-55 | 适当放宽直立判定，老人可能稍弯腰 |
| 跌倒 | `inactivity_frames` | 2 | 快速确认静止 |
| 跌倒 | `inactivity_threshold` | 20 | 适当放宽静止判定 |
| 跌倒 | `confirm_frames` | 1-2 | 最快响应 |
| 跌倒 | `cooldown` | 20 | 缩短冷却，多次跌倒需多次告警 |
| 徘徊 | `enabled` | `false` | 老人正常活动空间，不适用 |

---

### 场景四：银行 ATM / 高安全区域

重点关注徘徊和打架。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 3-4 | ATM 区域不应多人聚集 |
| 聚集 | `radius` | 150-200 | 标准窄角场景 |
| 聚集 | `cooldown` | 30 | 缩短冷却，持续警觉 |
| 打架 | `proximity_radius` | 100-130 | 近距离场景 |
| 打架 | `min_speed` | 60-80 | 降低阈值，敏感响应 |
| 打架 | `confirm_frames` | 3-4 | 快速确认 |
| 跌倒 | `confirm_frames` | 2-3 | 快速响应 |
| 徘徊 | `min_duration` | 30-45 | 正常取款 1-2 分钟，超时即可疑 |
| 徘徊 | `max_distance` | 80-100 | ATM 区域小 |
| 徘徊 | `max_displacement_ratio` | 0.2 | 严格判定来回踱步 |
| 徘徊 | `cooldown` | 60 | 缩短冷却 |

---

### 场景五：学校 / 操场

需平衡正常活动与异常行为。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 15-20 | 课间聚集是常态，只检测异常大规模 |
| 聚集 | `radius` | 300-400 | 操场广角 |
| 聚集 | `confirm_frames` | 10-15 | 长时间确认 |
| 打架 | `proximity_radius` | 120-150 | 中等距离 |
| 打架 | `min_speed` | 80-100 | 学生追逐打闹多，适度提高 |
| 打架 | `confirm_frames` | 5-8 | 适当增加减少打闹误报 |
| 跌倒 | `confirm_frames` | 3-5 | 标准值 |
| 跌倒 | `cooldown` | 60 | 运动场跌倒频繁，适当延长冷却 |
| 徘徊 | `enabled` | `false` | 学校场景通常不适用 |

---

### 场景六：健身房 / 运动场所

运动动作多，需大幅减少误报。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `enabled` | 视需求 | 团课期间关闭 |
| 打架 | `min_speed` | 120-150 | 运动速度本身很高 |
| 打架 | `confirm_frames` | 8-10 | 长时间确认 |
| 打架 | `joint_overlap_threshold` | 2-3 | 提高关节重叠要求 |
| 跌倒 | `ratio_threshold` | 1.2-1.5 | 俯卧撑等动作宽高比大 |
| 跌倒 | `min_ratio_change` | 0.8 | 提高变化阈值排除训练动作 |
| 跌倒 | `min_hip_velocity` | 50-60 | 排除缓慢躺下动作 |
| 跌倒 | `cooldown` | 120 | 训练时延长冷却 |
| 跌倒 | `inactivity_frames` | 5-8 | 需更长静止时间确认 |
| 徘徊 | `enabled` | `false` | 健身房长时间停留是正常行为 |

---

### 场景七：仓库 / 工厂车间

关注安全事故和违规进入。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 4-6 | 作业区不应过多人聚集 |
| 聚集 | `radius` | 200-300 | 工厂空间大，广角为主 |
| 打架 | `min_speed` | 80-100 | 标准阈值 |
| 跌倒 | `ratio_threshold` | 1.0-1.2 | 标准或略高 |
| 跌倒 | `min_hip_velocity` | 35-45 | 排除弯腰操作（工人常弯腰拾物） |
| 跌倒 | `confirm_frames` | 3-4 | 安全事故需及时响应 |
| 徘徊 | `min_duration` | 30-60 | 非作业人员不应长时间逗留 |
| 徘徊 | `max_distance` | 100-150 | 中等范围 |
| 徘徊 | `min_total_path` | 60-100 | 需有一定移动量 |
| 徘徊 | `cooldown` | 90 | 中等冷却 |

---

### 场景八：小区 / 围墙周界

夜间安防重点场景。

| 规则 | 参数 | 建议值 | 理由 |
|------|------|--------|------|
| 聚集 | `max_count` | 3-5 | 围墙边聚集高度可疑 |
| 聚集 | `radius` | 200-300 | 周界摄像头多为广角 |
| 聚集 | `cooldown` | 30 | 缩短冷却，持续监控 |
| 打架 | `proximity_radius` | 150-200 | 标准广角配置 |
| 打架 | `min_speed` | 60-80 | 需敏感捕捉 |
| 跌倒 | `enabled` | 视需求 | 周界关注入侵，跌倒次要 |
| 徘徊 | `min_duration` | 60-90 | 围墙边不应长时间逗留 |
| 徘徊 | `max_distance` | 120-200 | 沿围墙来回走 |
| 徘徊 | `max_displacement_ratio` | 0.25 | 较严格判定踩点行为 |
| 徘徊 | `cooldown` | 60 | 加强监控频率 |

## 七、全局检测参数

| 参数 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| `detect.fps` | DetectConfig | `5` | 检测帧率（1-30）。越高响应越快，但 CPU/GPU 负载越大 |
| `detect.confidence` | DetectConfig | `0.5` | YOLO 置信度阈值。降低检出更多人但增加误检 |
| `roi` | CameraConfig | `[]` | 感兴趣区域多边形。只有 ROI 内人员参与检测，为空则全画面检测 |
